HTML / CSS Boilerplate

2019-10-09T12:46:37.121Z

Here's some of the HTML/CSS code I include in every project.

HTML

<html lang="en">
  <head>
    <meta charset='utf-8' />
    <meta http-equiv='x-ua-compatible' content='ie=edge' />
    <meta name='viewport' content='width=device-width,initial-scale=1,shrink-to-fit=no' />

These meta tags will make your site look normal on mobile, specifically the last line.

<meta name="description" content="Your page description">
<title>Page Title</title>

At a minimum, you should have these tags for usability/SEO purposes.

CSS

*, ::after, ::before {
  box-sizing: border-box;
}

Without this, width and height don't include padding and border width. So if you were to set width = 100px without this setting, then your width would actually be 100px + padding + border. When most people set width = 100px, they want width = 100px. Why it doesn't default to this setting I have no idea.

html {
  font-size: 62.5%;
}

body {
  font-size: 1.6rem;
}

rem units enable one to size things relative to the font-size of html. rem units are generally preferrable to px or em because they scale relative to the global font-size (without the confusion of em depending on its parent).

1 rem = 10 * font-size of html

Making the html font-size 10px means that 1rem = 10px, making calculations easier. But it's not good practice to directly set the html font-size because that overrides the user's default browser font-size settings, so we set the font-size to a percentage. The default font-size is generally 16px, and 62.5% of 16px = 10px.

A font-size of 1.6rem sets the body font-size back to 100%, or 16px on most browsers (16px _ 62.5% _ 1.6 = 16px)

body {
  margin: 0;
  padding: 0;
}

Remove any default margin and padding in the body.

h1,h2,h3,h4,h5,h6 {
  margin-top: 0;
}

To be honest you should probably get rid of all the margins, but at the very least you should remove the top margin because it's totally unnecessary.

a {
  color: <anything else>
  text-decoration: none;
}

The default link color is atrocious. Remove the underline, and change the color to anything else.

@media (prefers-color-scheme: dark) {
  body {
    background-color: hsl(0, 0%, 15%);
    color: white;
  }
}

Actually I prefer to put dark mode in classes (and CSS variables) in order to allow the user to toggle between dark/light mode, but if you're just spinning something up quickly then this is the least you can do. And either way you should be reading the media query, even if via Javascript (see article for more).

There's a lot more CSS I include, this is just a barebones starting point.
